Green attempts to guess at the screen co-ordinates but relies on heuristics to try to convert it position:fixed values. Known sources of error mapping screen co-ordinates to the page include:
  • When embedded in an iframe, the frame's offset from the window
  • Pinch zoom / browser zoom scale
  • On mobile, the offset of the context in the screen (top controls, OS heading, etc.)
  • Mobile (but not desktop) Safari uses device pixels instead of CSS pixels (fails to divide by the divicePixelRatio)
